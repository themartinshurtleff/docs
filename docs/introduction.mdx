# Introduction

TradeNet Terminal is a native desktop trading terminal for crypto derivatives. Built in Rust. GPU-accelerated rendering via wgpu. Sub-1ms exchange latency. It aggregates order flow from Binance, Bybit, OKX, and Hyperliquid into a single interface.

Currently in closed beta (v0.8.6). Available on Windows and macOS.

## The Problem

Every existing terminal shows you what already happened. Liquidation maps show past liquidations. Heatmaps show historical clusters. You are always reacting. By the time you see the data, the move is over.

That is not good enough.

## The Solution

TradeNet Terminal runs a self-calibrating liquidation prediction algorithm. It does not show you where liquidations happened. It shows you where they WILL happen.

The algorithm ingests real-time order flow from all four supported exchanges, models open interest distribution, and projects liquidation clusters before price reaches them. You see the targets. You position ahead of the move.

## Supported Exchanges

- **Binance** - USDT and COIN-M perpetuals
- **Bybit** - Linear and inverse perpetuals
- **OKX** - USDT-M and COIN-M swaps
- **Hyperliquid** - All listed perpetuals

Data streams from all exchanges simultaneously. No API keys needed for market data. The terminal handles connection management and reconnection automatically.

## Tech Stack

The terminal is built from scratch. No Electron. No web wrapper.

- **Rust** core for performance and reliability
- **wgpu** for GPU-accelerated chart rendering
- **Hetzner backend** for server-side data processing and aggregation
- **Custom protocol** for compressed market data streaming

Everything is optimized for speed. The rendering pipeline runs on your GPU. Data processing runs on dedicated servers. Your machine handles display. The servers handle the math.
